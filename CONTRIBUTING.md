# 🤝 Contributing Guidelines

Thank you for your interest in contributing to the Azure Sentinel SOC Security Portfolio!

## How to Contribute

### 1. Fork the Repository
```bash
git clone https://github.com/your-username/Azure-Sentinel-SOC-Security-Portfolio.git
cd Azure-Sentinel-SOC-Security-Portfolio
git checkout -b feature/your-feature-name
```

### 2. Make Your Changes

- Add your KQL queries, scripts, or documentation
- Follow the existing code structure
- Add comments and documentation
- Test thoroughly

### 3. Commit Your Changes
```bash
git add .
git commit -m "Add: Description of your contribution"
```

### 4. Push and Create Pull Request
```bash
git push origin feature/your-feature-name
```

Create a PR with:
- Clear description
- Related issues
- Testing notes

## Contribution Types

### KQL Queries
- Test in your Sentinel environment
- Add MITRE ATT&CK mapping
- Include comments
- Document use case

### Python Scripts
- Follow PEP 8 style guide
- Include docstrings
- Add error handling
- Test with Azure SDK

### Documentation
- Use clear markdown
- Include examples
- Add links to resources
- Proofread carefully

## Code of Conduct

- Be respectful and inclusive
- No harassment or discrimination
- Focus on technical excellence
- Help others learn

## Questions?

Open an issue or discussion!
